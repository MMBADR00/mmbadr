CortexLab

Overview

This application is implemented using the Python programming language and employs the PySide6 library to build a flexible, modular, and user friendly Graphical User Interface (GUI). It integrates multiple specialized tools into a unified environment, each targeting different aspects of battery modeling, diagnosis, real-time monitoring, and performance evaluation.

  • Modular design that organizes battery workflows into independent tools under one desktop application
  • Consistent GUI layout and interaction patterns across tools to support clear navigation and usability
  • Workflows covering offline analysis, visualization, estimation tasks, and real-time monitoring
  • Machine learning workflows supporting model training, evaluation, and inference for prediction and classification

Tools

Math Studio

It is the general-purpose computation workspace. It combines a workspace environment, calculator tools, and matrix operations. The embedded workspace supports commands, variables, file operations, and plotting. It is used for custom mathematical analysis beyond predefined battery tools.

Supplementary Materials

It is the application reference resource. It is not an analysis tool; it is a curated documentation browser that opens bundled CortexLab documents, research data references, and downloadable code or data packages that support the other tools. It is intended for accessing manuals, research papers, datasets, and example repositories.

Battery Configuration

It is the pack topology builder. Series, parallel, or combined configurations are selected, and cell voltage and capacity are entered along with the number of cells connected in series and the number of cells connected in parallel. The tool computes pack-level voltage, capacity, and energy, and visualizes the electrical layout. It supports both numerical calculation and graphical explanation of pack architecture.

Battery Performance Metrics

It is a key performance indicator calculator rather than a full modeling pipeline. It evaluates operational metrics such as state-of-charge, state-of-power, state-of-energy, state-of-health, and remaining useful life. It presents equations, results tables, and visualizations. It is intended for quick evaluation of battery performance indicators.

OCV-SOC Modeling

It builds an open-circuit voltage to state-of-charge relationship from uploaded data. It supports selection of measurement and voltage columns, data inspection, cleaning of anomalies and missing values, and polynomial fitting with chi-squared error minimization. It also enables estimation of state-of-charge from a given open-circuit voltage value. It is used for constructing the static voltage to state-of-charge mapping.

Battery State Estimation

It is the estimation tool for recorded battery datasets. It supports loading comma-separated values or Excel files, data preview and cleaning, range selection, and execution of state-of-charge and state-of-health estimation workflows. Methods include Coulomb counting for state-of-charge estimation and capacity fade method for state-of-health estimation. It is intended for analysis of previously collected battery data.

Equivalent Circuit Modeling

It is the dynamic model identification tool. It identifies a first-order Thevenin equivalent circuit model using methods such as least squares, Levenberg-Marquardt, and recursive least squares. It also validates the model using filters such as extended Kalman filter, adaptive extended Kalman filter, unscented Kalman filter, adaptive unscented Kalman filter, and particle filter. It is used for parameter identification, terminal voltage reconstruction, and model validation.

Drive Cycle Generator

It builds a route-based electric vehicle drive cycle. Origin and destination coordinates are entered, along with Google Maps and OpenWeather application programming interface keys, and optional tuning of behavior factor and uphill tolerance. The outputs include a generated route dataset, speed profile, wind-compass plot, and embedded map. It is used to generate realistic driving profiles for battery studies.

Machine Learning Regression

It is an end-to-end regression pipeline. It supports data upload, preprocessing, feature and target selection, train and test splitting, model training, hyperparameter tuning, validation, evaluation metrics, visualization, and model export. It is used for continuous numerical prediction tasks such as state-of-charge estimation.

Machine Learning Classification

It is the classification counterpart to the regression tool. It follows a similar pipeline but focuses on classification models, evaluation metrics, and outputs such as confusion matrices and multi-class predictions. It is used when the target variable represents categorical labels such as fault detection or health classification.

Real-Time Monitoring

It is the live hardware interface tool. It connects to Arduino or Firmata, maps analog pins to voltage, current, and temperature sensors, and streams data in real time. It supports real-time battery state estimation and applies machine learning models for both estimation and classification on incoming data. It includes visualization dashboards for sensors, battery states, and data-driven outputs.
